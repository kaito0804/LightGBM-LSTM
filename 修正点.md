
# 1. AIの「過学習（Overfitting）」への懸念
「LightGBMとLSTMを使っていますが、入力データがRSIやMACDなどの一般的な指標ばかりです。これらが『ただのカーブフィッティング（過去のデータに無理やり合わせただけ）』になっていない証明はありますか？ 
ウォークフォワードテスト（期間をずらした検証）の結果を見せてください。」
 一般的な指標だけで勝てるなら誰も苦労しません。板情報（Orderbook）の厚みや、BTCとの相関など、もっと生々しいデータ入力の比重を高める必要があります。

# 2. AIとルールベースの矛盾
質問: 「improved_signal_scoring.py で人間が決めた固定ルール（RSIが30以下なら買い、など）が入っていますが、せっかくAIを使っているのに、なぜ人間が決めた古いルールで判断を上書きするのですか？ 
AIの判断精度に自信がないということですか？」
高額なAI botなのに、結局中身が「RSIが30だから買う」という単純なプログラムであれば、AIである意味がありません。

# 3. MLモデルの学習プロセスの開示
「ml_predictor.pyでモデルを定義していますが、実際にどのデータを教師データとして学習させたのか、その特徴量の重要度（Feature Importance）はどうなっているのかレポートをください。
単に価格とRSIを入れただけでは勝てないはずです。」
